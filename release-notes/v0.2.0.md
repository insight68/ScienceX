# ScienceX v0.2.0 — 自动化三平台发布 / Automated Three-Platform Release

---

## v0.2.0 的重点 / What's in v0.2.0

**中文：**

v0.2.0 是一个面向分发的基础设施版本。我们没有加入新的科研功能，而是把发布和质量保障打通：

- **自动化三平台发布**：推送 `v*` 标签后，GitHub Actions 在 macOS / Windows / Linux 三个平台上并行构建安装包并自动上传到 GitHub Releases。用户首次可以直接从 Releases 页面下载。
- **CI 质量门禁**：每个 PR 和 push 到 main 都会自动运行服务端测试、桌面端 lint+测试+构建、适配器测试，保持主分支稳定。
- **未签名构建的最佳实践**：在没有 Apple 开发者证书的前提下，macOS 构建关闭签名和公证，用户首次安装执行一次 `xattr -cr` 即可；Windows 出现 SmartScreen 提示时点击「仍要运行」。workflow 预留了签名/公证的环境变量入口，未来获得证书后只需配置 Secrets 即可切换到签名发布。

本次版本号从 0.1.0 直接跳到 0.2.0，期间 0.1.x ~ 0.4.x 为内部迭代版本，未对外发布。

---

**English:**

v0.2.0 is an infrastructure-focused release for distribution. We didn't add new research features — we wired up release and quality assurance:

- **Automated three-platform release**: pushing a `v*` tag triggers GitHub Actions to build installers in parallel on macOS / Windows / Linux and upload them to GitHub Releases. Users can download directly from the Releases page.
- **CI quality gate**: every PR and push to main automatically runs server tests, desktop lint+test+build, and adapter tests to keep the main branch stable.
- **Best practice for unsigned builds**: without an Apple Developer certificate, macOS builds disable signing and notarization — users run `xattr -cr` once on first install; Windows shows SmartScreen, click "More info" → "Run anyway". The workflow reserves environment variable entry points for signing/notarization — once a certificate is obtained, switching to signed releases is just a matter of configuring Secrets.

The version jumps from 0.1.0 to 0.2.0; the intermediate 0.1.x ~ 0.4.x were internal iterations, not publicly released.

## 首次安装说明 / First-Time Installation

### macOS

安装包未签名，系统会拦截。提示「已损坏」或「无法验证开发者」时，终端执行：

*The installer is unsigned; macOS will block it. When you see "damaged" or "cannot verify developer," run in terminal:*

```bash
xattr -cr /Applications/ScienceX.app
```

### Windows

未签名安装包会出现 SmartScreen 提示，点击「更多信息」→「仍要运行」即可。

*Unsigned installers trigger SmartScreen. Click "More info" → "Run anyway."*

### Linux

```bash
sudo dpkg -i sciencex_*.deb
# 或运行 AppImage
chmod +x ScienceX-*.AppImage
./ScienceX-*.AppImage
```

## 已知限制 / Known Limitations

- `table-quality-v1` 配方最多分析 100 个安全解析的样本行，**不构成完整数据集统计、显著性检验或科学结论** / The `table-quality-v1` recipe profiles at most 100 safely parsed sample rows — it is **not full-dataset statistics, significance testing, or a scientific conclusion**.
- 通用计算环境（Python / Jupyter / R）尚在开发中 / General compute environments (Python / Jupyter / R) are still under development.
- 科研连接器（文献、科学数据库、HPC）尚未就绪 / Scientific connectors (literature, databases, HPC) are not yet available.
- macOS Intel (x64) 包暂未提供 / macOS Intel (x64) build is not yet provided.
