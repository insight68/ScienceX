# ScienceX v0.1.0 — 首个公开发布 / First Public Release

## 为什么做 ScienceX / Why We Built ScienceX

**中文：**

2026 年6月，Anthropic 发布了 [Claude Science](https://www.anthropic.com/news/claude-science-ai-workbench) —— 一个面向科学研究的 AI 工作台，它将实验数据、可复现运行、Provenance 追踪和研究产物整合进统一环境。这个消息让我们非常兴奋：AI 辅助科研终于有了产品级的形态。但兴奋之余，我们也看到了一个根本性的问题：

**科学研究不应该被锁定在单一公司的订阅里。**

科学的本质是可复现、可审计、可验证。如果一个科研工作台的核心——数据存储在哪里、模型如何调用、运行是否可重放、产物是否可追溯——全部由一家公司控制，那它就不是真正的科学基础设施，而是一个科学外包服务。

这就是 ScienceX 的起点：**用开源的方式，做一个 Claude Science 的本地优先替代方案。**

我们的信念很简单：

- **数据主权**：实验表格、运行记录和研究产物应该留在研究者自己的机器或基础设施上，而不是上传到某个云服务。
- **模型中立**：研究者应该自由选择 Anthropic、OpenAI、DeepSeek、Ollama 或任何兼容 API —— 科研工作流不应该绑定到单一订阅。
- **不可篡改的可追溯性**：每一次运行都应留下完整的输入哈希、参数快照、环境信息、状态转换和产物哈希。科学要求可审计，AI 辅助科学更应如此。
- **开放扩展**：研究者已有的工具、脚本、数据库和计算资源，应该能通过 Skills、MCP、SubAgent 和终端无缝接入，而不是被迫迁移到某个封闭平台。

v0.1.0 是这条路上的第一步。它还不是 Claude Science 的功能对等替代——我们坦诚地说，当前版本只完成了科研工作台的基础框架：研究项目管理、实验表格登记、数据画像、可追溯 Run、Provenance 记录和 Artifacts 生成。下面的 Agent 基础设施——多模型、多会话、Skills、MCP、Computer Use、权限审批——已经打磨了相当长的时间，因为我们坚持认为，**没有扎实的 Agent 底座，科研工作台就是空中楼阁。**

ScienceX 不是 Anthropic 的产品，也不隶属于 Anthropic。它是一个独立的开源项目，由 [iteamify.com](https://iteamify.com) 维护。我们站在 Claude Science 的产品愿景上，用开源的方式重新实现它，并加入我们认为科学研究真正需要的原则：本地优先、模型中立、可审计、开放扩展。

这条路很长，但我们相信方向是对的。欢迎加入。

**English:**

In late 2025, Anthropic unveiled [Claude Science](https://www.anthropic.com/news/claude-science-ai-workbench) — an AI workbench purpose-built for scientific research, integrating experiment data, reproducible runs, provenance tracking, and research artifacts into a unified environment. We were genuinely excited: AI-assisted research finally had a product-shaped vision. But our excitement came with a fundamental concern:

**Scientific research should never be locked inside a single company's subscription.**

The essence of science is reproducibility, auditability, and verifiability. If the core of a research workbench — where data lives, how models are invoked, whether runs are replayable, whether artifacts are traceable — is entirely controlled by one vendor, it isn't scientific infrastructure. It's scientific outsourcing.

That's where ScienceX begins: **to build an open-source, local-first alternative to Claude Science.**

Our convictions are straightforward:

- **Data sovereignty**: experiment tables, run records, and research artifacts belong on the researcher's own machine or infrastructure — not uploaded to someone else's cloud.
- **Model neutrality**: researchers should freely choose between Anthropic, OpenAI, DeepSeek, Ollama, or any compatible API. The research workflow must never be tied to a single subscription.
- **Immutable traceability**: every run must capture complete input hashes, parameter snapshots, environment info, state transitions, and artifact hashes. Science demands auditability — AI-assisted science demands it even more.
- **Open extensibility**: existing tools, scripts, databases, and compute resources must connect through Skills, MCP, SubAgents, and terminals — not get abandoned for a closed platform.

v0.1.0 is the first step on this road. It is not yet a feature-complete equivalent of Claude Science — we'll be honest: this release delivers the foundational framework of a research workbench: project management, experiment-table registration, data profiling, traceable runs, provenance records, and artifact generation. The agent infrastructure underneath — multi-model, multi-session, Skills, MCP, Computer Use, permission review — has been battle-tested for far longer, because we firmly believe that **without a solid agent foundation, a research workbench is a castle built on sand.**

ScienceX is not an Anthropic product and is not affiliated with Anthropic. It is an independent open-source project maintained by [iteamify.com](https://iteamify.com). We stand on the product vision of Claude Science while reimagining it as open source, and we build into it the principles we believe scientific research truly needs: local-first, model-neutral, auditable, and openly extensible.

The road ahead is long, but the direction feels right. Join us.

## v0.1.0 包含的能力 / What's Included

### 科研工作台 / Science Workbench

| 能力 Capability                   | 描述 Description                                                                                                                                                         |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 研究项目管理 Research Projects    | 在本地目录创建`.sciencex` manifest 和 SQLite 研究数据库 / Creates `.sciencex` manifest and SQLite database in a local directory                                      |
| 实验表格登记 Table Registry       | 支持 UTF-8 CSV/TSV，记录规范化路径、大小、修改时间和 SHA-256 / Registers CSV/TSV with canonical path, size, mtime, and SHA-256                                           |
| 本地数据画像 Data Profiling       | 推断列类型，统计缺失值、唯一值、完整行和数值列（最多 100 样本行） / Infers column types, measures missing values, uniqueness, and numeric stats (up to 100 sampled rows) |
| 可追溯 Run Traceable Runs         | 显式`queued / running / completed / failed / interrupted` 状态机，记录参数和运行环境 / Explicit state machine with parameter and environment snapshots                 |
| Provenance 追踪 Provenance        | Append-only`events.jsonl`、Run manifest、输入哈希和配方哈希 / Append-only event log, run manifest, input hashes, and recipe hashes                                     |
| 研究产物 Artifacts                | 生成并登记`quality-report.md` 和 `profile.json`，含大小和内容哈希 / Registered artifacts with size and content hashes                                                |
| 重放与过期检测 Replay & Staleness | 历史 Run 可重放为子 Run；数据更新后旧 Run 标记为`stale` / History replay as child runs; old runs marked `stale` on new data versions                                 |

### Agent 基础设施 / Agent Infrastructure

- **多会话工作台 Multi-session workspace**: 标签页、项目切换、终端入口和会话历史集中管理 / Tabs, project switching, terminal entry, and session history in one place
- **代码编辑与 Diff Code editing & diff**: Markdown / 代码块 / Diff 视图优化渲染，Bash、Read、Edit、Write 等工具调用专属展示面板 / Optimized rendering for Markdown, code blocks, and diffs; dedicated panels for Bash, Read, Edit, Write tool calls
- **权限与确认流 Permission review**: 危险命令、工具调用和 AI 反问弹窗审批 / Approve risky commands, tool calls, and model follow-ups in modal dialogs
- **多模型提供商 Multi-model**: Anthropic 兼容 API、第三方模型、WebSearch fallback 和本地配置 / Anthropic-compatible APIs, third-party models, WebSearch fallback, and local config
- **Skills / MCP / SubAgent**: 可扩展能力插件、多代理编排与并行任务执行 / Extensible capability plugins, agent orchestration, and parallel tasks
- **Computer Use**: Agent 授权后截图、点击、输入并控制桌面应用 / Let the agent take screenshots, click, type, and control desktop apps
- **IM 接入 IM integration**: Telegram / 飞书远程对话、切换项目和审批权限 / Remote chat, project switching, and permission approval via Telegram / Feishu
- **定时任务 Scheduled tasks**: 创建计划任务，查看本机 Token 使用趋势 / Create planned tasks and track local token usage trends
- **H5 远程访问 H5 remote access**: 一次性令牌在手机或其他设备上接入当前会话 / Access the current session from a phone or another device with a one-time token

## 首次安装说明 / First-Time Installation

### macOS

安装包未经过开发者签名，系统会拦截运行。提示「已损坏」或「无法验证开发者」时，终端执行：

*The installer is not yet developer-signed; macOS will block execution. When you see "damaged" or "cannot verify developer," run in terminal:*

```bash
xattr -cr /Applications/ScienceX.app
```

### Windows

未签名安装包可能出现 SmartScreen 提示，点击「更多信息」→「仍要运行」即可。

*Unsigned installers may trigger SmartScreen. Click "More info" → "Run anyway."*

### Linux

```bash
sudo dpkg -i sciencex_*.deb
```

## 从源码运行 / Run from Source

适合想调试底层逻辑、参与开发或体验最新功能的用户：

*For users who want to debug, contribute, or try the latest features:*

```bash
git clone https://github.com/insight68/sciencex.git
cd sciencex
bun install

# 启动完整桌面端 / Launch the full desktop app
cd desktop
bun install
bun run build:sidecars
bun run electron:dev
```

打开桌面窗口后，从左侧导航进入 **Science**。项目创建、表格预览和 `table-quality-v1` 配方不需要模型 API Key；仅在使用 AI Agent 对话时才需配置提供商。

*Open **Science** from the left navigation after the window appears. Creating projects, previewing tables, and running `table-quality-v1` do not require a model API key — configure a provider only when using AI agent chat.*

## 已知限制 / Known Limitations

- `table-quality-v1` 配方最多分析 100 个安全解析的样本行，**不构成完整数据集统计、显著性检验或科学结论** / The `table-quality-v1` recipe profiles at most 100 safely parsed sample rows — it is **not full-dataset statistics, significance testing, or a scientific conclusion**.
- 通用计算环境（Python / Jupyter / R）尚在开发中 / General compute environments (Python / Jupyter / R) are still under development.
- 科研连接器（文献、科学数据库、HPC）尚未就绪 / Scientific connectors (literature, databases, HPC) are not yet available.
